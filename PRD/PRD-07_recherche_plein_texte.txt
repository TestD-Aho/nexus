# PRD-07 - Recherche plein texte etudiants avec laravel/scout

## Metadata
- Version: 1.0
- Date: 2026-03-14
- Dependances amont: PRD-02, PRD-03, PRD-04, PRD-05
- Composants imposes: `laravel/scout`, `symfony/string`

## Resume executif
Ce PRD introduit une recherche plein texte performante et securisee sur les listes etudiantes (et extensions futures), avec indexation progressive et controle strict des filtres metier.

## Objectifs
1. Offrir une recherche rapide sur etudiants et dossiers associes.
2. Supporter recherche multi-champs et typo tolerance minimale.
3. Maintenir securite d'acces (annee, role, permissions).
4. Assurer coherences des resultats avec les donnees courantes.

## Contraintes non negociables
1. Aucun changement schema BDD.
2. Les permissions existantes doivent filtrer les resultats.
3. Pas d'exposition de donnees sensibles hors scope utilisateur.
4. Mecanisme de reindexation sans interruption majeure.

## Portee fonctionnelle phase 1
- Recherche etudiants:
  - `num_carte_etud`
  - `nom_etu`
  - `prenom_etu`
  - `email_etu`
  - `promotion_etu`
- Filtres:
  - annee academique (`inscriptions.id_annee_acad`)
  - niveau (`inscriptions.id_niv_etude`)
  - statut parcours (selon contexte metier)

## Securite et filtrage
- Scope obligatoire par role:
  - admin: scope complet
  - scolarite: scope annee active + permissions
  - enseignant/commission: scope metier dedie
- Validation des filtres avant execution.
- Nettoyage requete utilisateur (string normalizer).

## Annexe detaillee G - specification index, scoring et securisation recherche

### Contrat document indexe (etudiant)
```json
{
  "id": "num_carte_etud",
  "num_carte_etud": "...",
  "nom_etu": "...",
  "prenom_etu": "...",
  "email_etu": "...",
  "promotion_etu": "...",
  "id_annee_acad": 0,
  "id_niv_etude": 0,
  "search_blob": "nom prenom email matricule"
}
```

### Politique de scoring
- Poids fort: `num_carte_etud` exact match.
- Poids moyen: `nom_etu`, `prenom_etu`.
- Poids faible: `email_etu`, `promotion_etu`.
- Bonus exactitude prefixe sur nom/prenom.

### Matrice de filtrage securite
| Profil | Scope recherche |
|---|---|
| Administrateur | global |
| Scolarite | annee active + permissions |
| Enseignant | scope pedagogique autorise |
| Commission | scope dossiers autorises |

### Reindexation et coherence
1. Reindexation initiale batch.
2. Reindexation incrementale sur ecritures repository.
3. Tache de reconciliation periodique index vs DB.
4. Alertes sur ecarts detectes.
